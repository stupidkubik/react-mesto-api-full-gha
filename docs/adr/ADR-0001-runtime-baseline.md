# ADR-0001: Runtime Baseline for Phase 1

## Status

Accepted

## Date

2026-03-10

## Context

Текущий репозиторий находится в переходном состоянии:

- frontend использует `react-scripts@5` (CRA)
- backend использует legacy CommonJS stack на `Express 4`
- CI все еще завязан на `Node 16.x`
- корневого runtime baseline для локальной разработки нет

Для `Phase 1` нужен единый runtime, который:

- достаточно современный для дальнейшей модернизации
- не форсирует немедленный big-bang апгрейд всех зависимостей
- подходит как переходный baseline для локальной разработки и будущего CI

## Decision

Для `Phase 1` зафиксировать runtime baseline как `Node 22.13.1` через `.nvmrc`.

Это переходное решение, а не финальный long-term target.

## Rationale

- `Node 16.x` больше не подходит как baseline модернизации.
- Сразу фиксировать `Node 24` для всего репозитория рискованно, пока frontend все еще держится на CRA и старом toolchain.
- `Node 22.13.1` уже доступен в текущем рабочем окружении, что снижает трение на старте `Phase 1`.
- Такой baseline позволяет сначала стабилизировать workspace, CI и контракты, а уже затем повторно оценить переход на `Node 24`.

## Consequences

Положительные:

- у репозитория появляется единый локальный runtime baseline
- можно синхронизировать локальную среду и новый CI
- снижается риск ранней несовместимости legacy frontend tooling

Отрицательные:

- это промежуточный, а не окончательный runtime target
- позже потребуется отдельная проверка и, вероятно, отдельный PR для перехода на `Node 24`

## Follow-up

Пересмотреть baseline после выполнения двух условий:

1. frontend больше не зависит от `react-scripts`
2. новый CI и root tooling уже стабилизированы
